1. Basic Class Structure & Main Method

What is a Class?

A class is a blueprint or template that defines the structure and behavior (properties and methods) of objects. It's the fundamental building block of Java programs.

Basic Class Syntax

public class MyClass {
    public static void main(String[] args) {
        System.out.println("Hello, World!");
    }
}

Understanding public static void main(String[] args)

This is the exact signature the JVM (Java Virtual Machine) looks for to start your program. Every part is mandatory.

Why public?

Makes the method accessible to the JVM, which is outside your class. Without public, the JVM cannot access the method to start your program.

Why static?

Allows the method to run without creating an object of the class. The JVM calls main directly on the class itself.

Why void?

Specifies that main does not return any value to the JVM.

Why String[] args?

It's an array of String objects. Receives command-line arguments when running the program. If no arguments are provided, the array is empty but not null.

📌 Critical Distinction: Empty array (length 0, exists in memory) vs Null array (no array object exists). 
// Empty array - array EXISTS with 0 elements
String[] arr = new String[0];
System.out.println(arr.length);  // 0

// Null array - NO array object exists
String[] arr2 = null;
System.out.println(arr2 == null);  // true

2. Loops in Java

Loops are used to execute a block of code repeatedly based on a condition. Java provides four types of loops.

1. For Loop

Used for: When you know the number of iterations in advance.

Basic For Loop

for (int i = 0; i < 5; i++) {
    System.out.println("Iteration: " + i);
}

Enhanced For Loop (For-Each)

Used to iterate over arrays and collections without index management.

int[] numbers = {10, 20, 30, 40};

for (int num : numbers) {
    System.out.println(num);
}

// Iterating over ArrayList
ArrayList<String> fruits = new ArrayList<>();
fruits.add("Apple");
fruits.add("Banana");

for (String fruit : fruits) {
    System.out.println(fruit);
}

2. While Loop

Used for: When you don't know the number of iterations. Continues while condition is true.

int i = 0;
while (i < 5) {
    System.out.println("Value: " + i);
    i++;
}
⚠️ Warning: If the condition is always true, you get an infinite loop. Make sure the condition will eventually become false.

3. Do-While Loop

Used for: When you want to execute code at least once, even if condition is false. Condition is checked at the end.

int i = 0;
do {
    System.out.println("Value: " + i);
    i++;
} while (i < 5);

4. Loop Control Statements

Break Statement

for (int i = 0; i < 10; i++) {
    if (i == 5) {
        break;  // Exits loop when i equals 5
    }
    System.out.println(i);
}

Continue Statement

for (int i = 0; i < 5; i++) {
    if (i == 2) {
        continue;  // Skips when i equals 2
    }
    System.out.println(i);
}

Loop Comparison Table

Loop Type When to Use Condition Checked
For Known number of iterations Before each iteration
For-Each Iterate over collections/arrays Automatically managed
While Unknown iterations, condition-based Before each iteration
Do-While Must execute at least once After each iteration

3. Arrays in Java

What are Arrays?

Arrays are reference types that store multiple values of the same data type in a single variable. They have a fixed size once created and elements are accessed by index (0-based).

Array Initialization

int[] numbers = new int[5];
int[] numbers2 = {10, 20, 30};
int[] numbers3 = new int[]{1, 2, 3};

Are Arrays Mutable?

Yes, arrays are mutable - you can change their elements.

int[] numbers = {1, 2, 3};
numbers[0] = 10;  // Modifying

Important: You cannot change the array's size after creation.

4. Object-Oriented Programming (OOP)

Java OOP is built on four pillars: Encapsulation, Inheritance, Abstraction, and Polymorphism.

1. Encapsulation

Definition: Wrapping data (variables) and code (methods) together. Variables are kept private and accessed through public getter/setter methods.

2. Inheritance

Definition: One class (child) acquires properties and methods from another class (parent).

3. Abstraction

Definition: Hiding complex implementation details and showing only essential features.

4. Polymorphism

Covered in next section with detailed examples.

5. Polymorphism (Overloading & Overriding)

Definition: Ability of an object to take many forms. Two types: Compile-time (Overloading) and Runtime (Overriding).

1. Method Overloading (Compile-time Polymorphism)

Definition: Multiple methods with the same name but different parameters (number, type, or order).

Example: Different Number of Parameters

class Calculator {
    public int add(int a, int b) {
        return a + b;
    }
    
    public int add(int a, int b, int c) {
        return a + b + c;
    }
    
    public double add(double a, double b) {
        return a + b;
    }
}

Calculator calc = new Calculator();
System.out.println(calc.add(5, 10));        // 15
System.out.println(calc.add(5, 10, 15));   // 30
System.out.println(calc.add(5.5, 10.2));   // 15.7

2. Method Overriding (Runtime Polymorphism)

Definition: Child class provides a specific implementation of a method that is already defined in its parent class.

Example

class Animal {
    void makeSound() {
        System.out.println("Some generic sound");
    }
}

class Dog extends Animal {
    @Override
    void makeSound() {
        System.out.println("Woof!");
    }
}

Animal dog = new Dog();
dog.makeSound();  // Output: Woof!

Overloading vs Overriding

Feature Overloading Overriding
Type Compile-time Runtime
Parameters Must differ Must be identical
Inheritance Not required Required

6. Abstract Classes vs Interfaces

Abstract Class

Definition: Cannot be instantiated. Can contain both abstract methods and concrete methods.

Interface

Definition: Blueprint containing only abstract methods and constants. Defines WHAT, not HOW.

Key Differences

Feature Abstract Class Interface
Inheritance Single (extends) Multiple (implements)
Methods Abstract + Concrete Abstract only
Constructors Yes No
Use Case "is-a" relationship "can-do" capability

7. Enums in Java

Definition: A special data type that allows you to define a set of constants (fixed values). Enums restrict variables to only predefined values.

Basic Enum Definition

enum Day {
    MONDAY, TUESDAY, WEDNESDAY, THURSDAY, FRIDAY, SATURDAY, SUNDAY
}

Day today = Day.MONDAY;
System.out.println(today);  // Output: MONDAY

Enum with Methods and Properties

enum Status {
    PENDING("Pending", 1),
    ACTIVE("Active", 2),
    INACTIVE("Inactive", 3);
    
    String displayName;
    int code;
    
    Status(String displayName, int code) {
        this.displayName = displayName;
        this.code = code;
    }
    
    public String getDisplayName() {
        return displayName;
    }
}

Enum in Switch Statements

Status status = Status.ACTIVE;

switch (status) {
    case PENDING:
        System.out.println("Waiting for approval");
        break;
    case ACTIVE:
        System.out.println("Currently active");
        break;
    case INACTIVE:
        System.out.println("No longer active");
        break;
}

10. Strings and Mutability

Are Strings Mutable?

Answer: IMMUTABLE - Once created, content cannot be changed.

String str = "Hello";

Are Arrays Mutable?

Answer: MUTABLE - You can change elements, but not size.

int[] arr = {1, 2, 3};

11. Errors & Exceptions - In Depth

Difference Between Error and Exception

Feature Error Exception
Recovery Not recoverable Recoverable with try-catch
Examples OutOfMemoryError, StackOverflowError IOException, NumberFormatException
Handling Cannot catch/handle Can catch/handle with try-catch-finally

Checked vs Unchecked Exceptions

Feature Checked Unchecked
Detection Compile time Runtime
Handling Required Yes (try-catch or throws) Not required (but recommended)
Examples IOException, SQLException NullPointerException, ArithmeticException

Try-Catch-Finally Execution Flow

Key Question: When an exception is caught, does remaining code execute or stop?

Answer:

  • Remaining code in try block is SKIPPED (after exception)
  • Catch block EXECUTES
  • Finally block ALWAYS executes (even if exception occurs)
  • Code AFTER try-catch-finally CONTINUES normally

Detailed Example

public class ExceptionDemo {
    public static void main(String[] args) {
        System.out.println("1. Start");
        
        try {
            System.out.println("2. Inside try");
            int result = 10 / 0;  // Exception
            System.out.println("3. SKIPPED");
        } catch (ArithmeticException e) {
            System.out.println("4. Caught exception");
        } finally {
            System.out.println("5. Finally - ALWAYS");
        }
        
        System.out.println("6. After try-catch-finally");
    }
}

Try-With-Resources

Auto-closes resources (files, connections) automatically.

try (FileReader fr = new FileReader("file.txt")) {
    // File automatically closed after try block
} catch (IOException e) {
    System.out.println("Error: " + e.getMessage());
}

12. File Handling in Java

Reading Files - Method 1: BufferedReader

try (BufferedReader br = new BufferedReader(
        new FileReader("test.txt"))) {
    
    String line;
    while ((line = br.readLine()) != null) {
        System.out.println(line);
    }
} catch (IOException e) {
    System.out.println("Error: " + e.getMessage());
}

Reading Files - Method 2: Files.readAllLines (Java NIO)

try {
    List<String> lines = Files.readAllLines(
        Paths.get("test.txt"));
    
    for (String line : lines) {
        System.out.println(line);
    }
} catch (IOException e) {
    System.out.println("Error: " + e.getMessage());
}

Writing Files

try (BufferedWriter bw = new BufferedWriter(
        new FileWriter("output.txt"))) {
    
    bw.write("Hello, World!");
    bw.newLine();
    bw.write("This is line 2");
    
} catch (IOException e) {
    System.out.println("Error: " + e.getMessage());
}

Appending to Files

try (BufferedWriter bw = new BufferedWriter(
        new FileWriter("output.txt", true))) {
    // true = append mode
    bw.newLine();
    bw.write("Appended content");
} catch (IOException e) {
    e.printStackTrace();
}

13. Collections Framework - Complete Deep Dive

Definition: Unified architecture of classes and interfaces to store and manipulate groups of objects efficiently.

Collections Hierarchy

Collection (Interface)
    ├── List (ordered, allows duplicates)
    │   ├── ArrayList
    │   ├── LinkedList
    │   └── Vector
    │
    ├── Set (unique elements)
    │   ├── HashSet
    │   ├── TreeSet
    │   └── LinkedHashSet
    │
    └── Queue (FIFO processing)
        ├── PriorityQueue
        └── LinkedList

Map (separate hierarchy - not part of Collection)
    ├── HashMap
    ├── TreeMap
    ├── LinkedHashMap
    └── ConcurrentHashMap

1. LIST Implementation - ArrayList

Characteristics: Ordered, allows duplicates, fast random access O(1), slower insertion/deletion O(n)

import java.util.*;

List<String> fruits = new ArrayList<>();

fruits.add("Apple");
fruits.add("Banana");
fruits.add("Apple");  // Duplicates allowed

System.out.println(fruits);  // [Apple, Banana, Apple]
System.out.println(fruits.get(0));  // Apple
System.out.println(fruits.size());  // 3

fruits.remove(1);
System.out.println(fruits.contains("Apple"));  // true

2. LIST Implementation - LinkedList

Characteristics: Ordered, allows duplicates, slow random access O(n), fast insertion/deletion at ends O(1)

LinkedList<Integer> numbers = new LinkedList<>();

numbers.add(10);
numbers.add(20);
numbers.add(30);

numbers.add(1, 15);  // Insert at index 1
numbers.addFirst(5);   // Add at beginning
numbers.addLast(40);   // Add at end

System.out.println(numbers.getFirst());  // 5
System.out.println(numbers.getLast());   // 40
numbers.removeFirst();
numbers.removeLast();

3. SET Implementation - HashSet

Characteristics: Unordered, no duplicates, O(1) operations

Set<String> colors = new HashSet<>();

colors.add("Red");
colors.add("Green");
colors.add("Blue");
colors.add("Red");  // Duplicate ignored

System.out.println(colors);  // Order not guaranteed
System.out.println(colors.contains("Red"));  // true
colors.remove("Green");

4. SET Implementation - TreeSet

Characteristics: Sorted automatically, no duplicates, O(log n) operations

Set<Integer> numbers = new TreeSet<>();

numbers.add(50);
numbers.add(10);
numbers.add(30);
numbers.add(20);

System.out.println(numbers);  // [10, 20, 30, 50] (sorted)
System.out.println(((TreeSet<Integer>)numbers).first());  // 10
System.out.println(((TreeSet<Integer>)numbers).last());   // 50

5. SET Implementation - LinkedHashSet

Characteristics: Maintains insertion order, no duplicates, O(1) operations

Set<String> linkedSet = new LinkedHashSet<>();

linkedSet.add("First");
linkedSet.add("Second");
linkedSet.add("Third");

System.out.println(linkedSet);  // [First, Second, Third]

6. QUEUE Implementation - PriorityQueue

Characteristics: Elements ordered by priority, removes lowest priority first

Queue<Integer> pq = new PriorityQueue<>();

pq.offer(50);
pq.offer(10);
pq.offer(30);
pq.offer(20);

while (!pq.isEmpty()) {
    System.out.println(pq.poll());  // Output: 10, 20, 30, 50
}

7. MAP Implementation - HashMap

Characteristics: Unordered, unique keys, O(1) operations, allows null key

Map<String, Integer> ages = new HashMap<>();

ages.put("Alice", 25);
ages.put("Bob", 30);
ages.put("Alice", 26);  // Updates value

System.out.println(ages.get("Alice"));  // 26
System.out.println(ages.containsKey("Bob"));  // true

// Iterating over entries (most efficient)
for (Map.Entry<String, Integer> entry : ages.entrySet()) {
    System.out.println(entry.getKey() + " -> " + entry.getValue());
}

8. MAP Implementation - TreeMap

Characteristics: Sorted by keys, unique keys, O(log n) operations

Map<String, Integer> treeMap = new TreeMap<>();

treeMap.put("Charlie", 28);
treeMap.put("Alice", 25);
treeMap.put("Bob", 30);

System.out.println(treeMap);  // {Alice=25, Bob=30, Charlie=28}

9. MAP Implementation - LinkedHashMap

Characteristics: Maintains insertion order, unique keys, O(1) operations

Map<String, String> linkedMap = new LinkedHashMap<>();

linkedMap.put("One", "First");
linkedMap.put("Two", "Second");
linkedMap.put("Three", "Third");

System.out.println(linkedMap);  // {One=First, Two=Second, Three=Third}

10. MAP Implementation - ConcurrentHashMap (Thread-safe)

Characteristics: Thread-safe without locking entire map, O(1) operations

import java.util.concurrent.ConcurrentHashMap;

ConcurrentHashMap<String, String> concurrentMap = 
    new ConcurrentHashMap<>();

concurrentMap.put("Java", "Programming Language");
concurrentMap.put("Python", "Scripting Language");

// Safe operations for concurrent access
concurrentMap.putIfAbsent("Go", "Systems Language");

11. Collections Utility Methods

List<Integer> numbers = new ArrayList<>(
    Arrays.asList(50, 10, 40, 20, 30));

// Sort ascending
Collections.sort(numbers);
System.out.println(numbers);  // [10, 20, 30, 40, 50]

// Sort descending
Collections.sort(numbers, Collections.reverseOrder());
System.out.println(numbers);  // [50, 40, 30, 20, 10]

// Reverse
Collections.reverse(numbers);

// Max and Min
System.out.println(Collections.max(numbers));  // 50
System.out.println(Collections.min(numbers));  // 10

// Binary search (list must be sorted)
Collections.sort(numbers);
int index = Collections.binarySearch(numbers, 30);
System.out.println("Index of 30: " + index);  // 2

// Frequency
List<String> words = new ArrayList<>(
    Arrays.asList("apple", "banana", "apple"));
System.out.println(Collections.frequency(words, "apple"));  // 2

12. Collections Comparison Table

Collection Duplicates Ordered Access Speed Best For
ArrayList Yes Yes O(1) Fast reading
LinkedList Yes Yes O(n) Fast insert/delete at ends
HashSet No No O(1) Unique elements, fast
TreeSet No Yes O(log n) Sorted unique elements
HashMap No (keys) No O(1) Fast key-value lookup
TreeMap No (keys) Yes O(log n) Sorted key-value pairs
📌 Collections Best Practices for SDET:
  • ArrayList: Use for sequential test data, frequent iteration
  • HashMap: Use for storing test results by test case ID
  • TreeSet: Use for sorted unique test result codes
  • ConcurrentHashMap: Use in multi-threaded test scenarios
  • Always specify generic types: <String>, <Integer>
  • Use entrySet() for iteration: Most efficient for Maps
