1. Basic Class Structure & Main Method

What is a Class?

A class is a blueprint or template that defines the structure and behavior (properties and methods) of objects. It's the fundamental building block of Java programs.

Basic Class Syntax

public class MyClass {
    public static void main(String[] args) {
        System.out.println("Hello, World!");
    }
}

Understanding public static void main(String[] args)

This is the exact signature the JVM (Java Virtual Machine) looks for to start your program. Every part is mandatory.

Why public?

Makes the method accessible to the JVM, which is outside your class. Without public, the JVM cannot access the method to start your program.

Why static?

Allows the method to run without creating an object of the class. The JVM calls main directly on the class itself.

Why void?

Specifies that main does not return any value to the JVM.

Why String[] args?

It's an array of String objects. Receives command-line arguments when running the program. If no arguments are provided, the array is empty but not null.

üìå Critical Distinction: Empty array (length 0, exists in memory) vs Null array (no array object exists). 
// Empty array - array EXISTS with 0 elements
String[] arr = new String[0];
System.out.println(arr.length);  // 0

// Null array - NO array object exists
String[] arr2 = null;
System.out.println(arr2 == null);  // true

2. Loops in Java

Loops are used to execute a block of code repeatedly based on a condition. Java provides four types of loops.

1. For Loop

Used for: When you know the number of iterations in advance.

Basic For Loop

for (int i = 0; i < 5; i++) {
    System.out.println("Iteration: " + i);
}

Syntax Breakdown:

  • Initialization: int i = 0 - runs once before loop starts
  • Condition: i < 5 - checked before each iteration
  • Increment: i++ - executed after each iteration

Enhanced For Loop (For-Each)

Used to iterate over arrays and collections without index management.

int[] numbers = {10, 20, 30, 40};

for (int num : numbers) {
    System.out.println(num);
}

// Iterating over ArrayList
ArrayList<String> fruits = new ArrayList<>();
fruits.add("Apple");
fruits.add("Banana");

for (String fruit : fruits) {
    System.out.println(fruit);
}

Advantages: Cleaner syntax, no index management, fewer errors.

2. While Loop

Used for: When you don't know the number of iterations. Continues while condition is true.

int i = 0;
while (i < 5) {
    System.out.println("Value: " + i);
    i++;
}
‚ö†Ô∏è Warning: If the condition is always true, you get an infinite loop. Make sure the condition will eventually become false.

3. Do-While Loop

Used for: When you want to execute code at least once, even if condition is false. Condition is checked at the end.

int i = 0;
do {
    System.out.println("Value: " + i);
    i++;
} while (i < 5);

Key Difference: Do-While loop body executes at least once; While loop may not execute at all if condition is false from start.

4. Nested Loops

A loop inside another loop. Commonly used for printing patterns or 2D arrays.

for (int i = 1; i <= 3; i++) {
    for (int j = 1; j <= 3; j++) {
        System.out.print(i + "," + j + " ");
    }
    System.out.println();
}

Output:

1,1 1,2 1,3
2,1 2,2 2,3
3,1 3,2 3,3

Loop Control Statements

Break Statement

Exits the loop immediately when a condition is met.

for (int i = 0; i < 10; i++) {
    if (i == 5) {
        break;  // Exits loop when i equals 5
    }
    System.out.println(i);
}

Continue Statement

Skips the current iteration and moves to the next one.

for (int i = 0; i < 5; i++) {
    if (i == 2) {
        continue;  // Skips when i equals 2
    }
    System.out.println(i);
}

Output: 0, 1, 3, 4 (skips 2)

Loop Comparison Table

Loop Type When to Use Condition Checked
For Known number of iterations Before each iteration
For-Each Iterate over collections/arrays Automatically managed
While Unknown iterations, condition-based Before each iteration
Do-While Must execute at least once After each iteration

3. Arrays in Java

What are Arrays?

Arrays are reference types that store multiple values of the same data type in a single variable. They have a fixed size once created and elements are accessed by index (0-based).

Array Declaration Forms

int[] arr;
String[] names;

Array Initialization

int[] numbers = new int[5];
int[] numbers2 = {10, 20, 30};
int[] numbers3 = new int[]{1, 2, 3};

Are Arrays Mutable?

Yes, arrays are mutable - you can change their elements.

int[] numbers = {1, 2, 3};
numbers[0] = 10;  // Modifying

Important: You cannot change the array's size after creation.

4. Object-Oriented Programming (OOP)

Java OOP is built on four pillars: Encapsulation, Inheritance, Abstraction, and Polymorphism.

1. Encapsulation

Definition: Wrapping data (variables) and code (methods) together. Variables are kept private and accessed through public getter/setter methods.

class Person {
    private String name;
    private int age;

    public String getName() {
        return name;
    }

    public void setName(String name) {
        this.name = name;
    }
}

2. Inheritance

Definition: One class (child) acquires properties and methods from another class (parent).

class Animal {
    void eat() { }
}

class Dog extends Animal {
    void bark() { }
}

3. Abstraction

Definition: Hiding complex implementation details and showing only essential features.

5. Polymorphism (Overloading & Overriding)

Definition: Ability of an object to take many forms. Two types: Compile-time (Overloading) and Runtime (Overriding).

1. Method Overloading (Compile-time Polymorphism)

Definition: Multiple methods with the same name but different parameters (number, type, or order).

Example 1: Different Number of Parameters

class Calculator {
    // Method 1: Two int parameters
    public int add(int a, int b) {
        return a + b;
    }
    
    // Method 2: Three int parameters
    public int add(int a, int b, int c) {
        return a + b + c;
    }
    
    // Method 3: Two double parameters
    public double add(double a, double b) {
        return a + b;
    }
}

public class Main {
    public static void main(String[] args) {
        Calculator calc = new Calculator();
        
        System.out.println(calc.add(5, 10));        // Calls Method 1: 15
        System.out.println(calc.add(5, 10, 15));   // Calls Method 2: 30
        System.out.println(calc.add(5.5, 10.2));   // Calls Method 3: 15.7
    }
}

Example 2: Different Parameter Order

class StringProcessor {
    public String combine(String a, int b) {
        return a + b;
    }
    
    public String combine(int a, String b) {
        return a + b;
    }
}

StringProcessor sp = new StringProcessor();
System.out.println(sp.combine("Value: ", 10));  // Value: 10
System.out.println(sp.combine(10, " is the value"));  // 10 is the value
üìå Key Points about Overloading:
  • Resolved at compile time
  • Return type CANNOT be different (must differ in parameters)
  • Parameters must differ in number, type, or order
  • Can have different access modifiers and exceptions

2. Method Overriding (Runtime Polymorphism)

Definition: Child class provides a specific implementation of a method that is already defined in its parent class.

Example

class Animal {
    void makeSound() {
        System.out.println("Some generic sound");
    }
}

class Dog extends Animal {
    @Override
    void makeSound() {
        System.out.println("Woof!");
    }
}

class Cat extends Animal {
    @Override
    void makeSound() {
        System.out.println("Meow!");
    }
}

public class Main {
    public static void main(String[] args) {
        Animal dog = new Dog();      // Parent reference, child object
        Animal cat = new Cat();
        
        dog.makeSound();  // Output: Woof! (Runtime decision)
        cat.makeSound();  // Output: Meow! (Runtime decision)
    }
}
üìå Key Points about Overriding:
  • Resolved at runtime (method called depends on actual object type)
  • Method signature must be exactly the same
  • Cannot have more restrictive access modifier
  • Cannot throw broader checked exceptions
  • Use @Override annotation (recommended)

Overloading vs Overriding

Feature Overloading Overriding
Type Compile-time Runtime
Method Name Same Same
Parameters Must differ Must be identical
Return Type Can differ (but not alone) Same or covariant
Inheritance Not required (same class) Required (parent-child)
Relationship Horizontal Vertical

Why Use Parent Reference with Child Object?

Question: Why use Animal animal = new Dog() instead of Dog dog = new Dog()?

Answer: Both are valid! Parent reference enables flexibility and polymorphism.

// Without polymorphism (Inflexible)
Dog dog = new Dog();
Cat cat = new Cat();
dog.makeSound();
cat.makeSound();

// With polymorphism (Flexible)
Animal[] animals = {new Dog(), new Cat()};
for (Animal animal : animals) {
    animal.makeSound();  // Calls correct method for each
}

Benefits of Polymorphism:

  • One method works with multiple types
  • Easier to add new types without changing existing code
  • Loose coupling between classes

6. Abstract Classes vs Interfaces

Abstract Class

Definition: Cannot be instantiated. Can contain both abstract methods and concrete methods.

abstract class Vehicle {
    String brand;
    
    public Vehicle(String brand) {
        this.brand = brand;
    }
    
    public void start() {
        System.out.println(brand + " starting...");
    }
    
    abstract void accelerate();
}

Interface

Definition: Blueprint containing only abstract methods and constants. Defines WHAT, not HOW.

interface Flyable {
    void fly();
}

interface Swimmable {
    void swim();
}

class Duck implements Flyable, Swimmable {
    public void fly() { }
    public void swim() { }
}

Key Differences

Feature Abstract Class Interface
Inheritance Single (extends) Multiple (implements)
Methods Abstract + Concrete Abstract only
Constructors Yes No
Use Case "is-a" relationship "can-do" capability

7. Enums in Java

Definition: A special data type that allows you to define a set of constants (fixed values). Enums restrict variables to only predefined values.

Why Use Enums?

  • Restrict variables to a fixed set of values
  • Improve code readability and maintainability
  • Prevent invalid values
  • Type-safe way to handle constants

Basic Enum Definition

enum Day {
    MONDAY, TUESDAY, WEDNESDAY, THURSDAY, FRIDAY, SATURDAY, SUNDAY
}

public class Main {
    public static void main(String[] args) {
        Day today = Day.MONDAY;
        System.out.println(today);  // Output: MONDAY
    }
}

Enum with Methods and Properties

enum Status {
    PENDING("Pending", 1),
    ACTIVE("Active", 2),
    INACTIVE("Inactive", 3);
    
    String displayName;
    int code;
    
    // Constructor (must be private)
    Status(String displayName, int code) {
        this.displayName = displayName;
        this.code = code;
    }
    
    public String getDisplayName() {
        return displayName;
    }
    
    public int getCode() {
        return code;
    }
}

public class Main {
    public static void main(String[] args) {
        Status status = Status.ACTIVE;
        System.out.println(status.getDisplayName());  // Output: Active
        System.out.println(status.getCode());           // Output: 2
    }
}

Iterating Over Enum Values

for (Day day : Day.values()) {
    System.out.println(day);
}

Enum in Switch Statements

Status status = Status.ACTIVE;

switch (status) {
    case PENDING:
        System.out.println("Waiting for approval");
        break;
    case ACTIVE:
        System.out.println("Currently active");
        break;
    case INACTIVE:
        System.out.println("No longer active");
        break;
}
üìå Enum Best Practices:
  • Use enums for fixed sets of constants
  • Enum constants are public, static, final by default
  • Constructors must be private
  • Perfect for status codes, options, and choices

Try-Catch-Finally Execution Flow

Key Question: When an exception is caught, does remaining code execute or stop?

Answer:

  • Remaining code in try block is SKIPPED (after exception)
  • Catch block EXECUTES
  • Finally block ALWAYS executes (even if exception occurs)
  • Code AFTER try-catch-finally CONTINUES normally

Detailed Example

public class ExceptionDemo {
    public static void main(String[] args) {
        System.out.println("1. Start");
        
        try {
            System.out.println("2. Inside try");
            int result = 10 / 0;  // Exception occurs
            System.out.println("3. SKIPPED - won't execute");
        } catch (ArithmeticException e) {
            System.out.println("4. Caught: " + e.getMessage());
        } finally {
            System.out.println("5. Finally - ALWAYS executes");
        }
        
        System.out.println("6. After try-catch-finally");
    }
}

Output:

1. Start
2. Inside try
4. Caught: / by zero
5. Finally - ALWAYS executes
6. After try-catch-finally

Multiple Catch Blocks

try {
    // code that might throw exceptions
} catch (ArithmeticException e) {
    System.out.println("Arithmetic error");
} catch (ArrayIndexOutOfBoundsException e) {
    System.out.println("Array index error");
} catch (Exception e) {
    System.out.println("Generic error");
} finally {
    System.out.println("Cleanup code");
}
üìå Exception Handling Best Practices:
  • Catch specific exceptions first, generic last
  • Always use finally for cleanup (closing files, resources)
  • Don't leave catch blocks empty
  • Log exceptions for debugging
  • Use try-with-resources for auto-closing resources

12. File Handling in Java

Why File Handling?

File handling is essential for reading, writing, and manipulating files. Important for SDET as test data often comes from files.

File Class

Definition: Represents a file or directory path (both exist).

import java.io.File;

public class FileDemo {
    public static void main(String[] args) {
        File file = new File("test.txt");
        
        System.out.println("Exists: " + file.exists());
        System.out.println("Is File: " + file.isFile());
        System.out.println("Is Directory: " + file.isDirectory());
        System.out.println("Path: " + file.getAbsolutePath());
        System.out.println("Size: " + file.length() + " bytes");
    }
}

Reading Files

Method 1: Using FileReader and BufferedReader

import java.io.*;

public class ReadFileDemo {
    public static void main(String[] args) {
        try (BufferedReader br = new BufferedReader(
                new FileReader("test.txt"))) {
            
            String line;
            while ((line = br.readLine()) != null) {
                System.out.println(line);
            }
        } catch (IOException e) {
            System.out.println("Error reading file: " + e.getMessage());
        }
    }
}

Method 2: Using Scanner

import java.io.*;
import java.util.Scanner;

public class ReadWithScanner {
    public static void main(String[] args) {
        try (Scanner scanner = new Scanner(
                new File("test.txt"))) {
            
            while (scanner.hasNextLine()) {
                String line = scanner.nextLine();
                System.out.println(line);
            }
        } catch (FileNotFoundException e) {
            System.out.println("File not found: " + e.getMessage());
        }
    }
}

Method 3: Using Files.readAllLines (Java NIO)

import java.nio.file.*;
import java.util.List;

public class ReadWithNIO {
    public static void main(String[] args) {
        try {
            List<String> lines = Files.readAllLines(
                Paths.get("test.txt"));
            
            for (String line : lines) {
                System.out.println(line);
            }
        } catch (IOException e) {
            System.out.println("Error: " + e.getMessage());
        }
    }
}

List Implementation Details

ArrayList

Characteristics: Ordered, duplicates allowed, fast random access (O(1)), slower insertion/deletion

Use when: Frequent reading, infrequent modifications

LinkedList

Characteristics: Ordered, duplicates allowed, slow random access (O(n)), fast insertion/deletion at ends (O(1))

Use when: Frequent insertions/deletions at ends

Set Implementation Details

HashSet

Characteristics: Unordered, no duplicates, O(1) operations

Use when: Need unique elements, order doesn't matter

TreeSet

Characteristics: Sorted, no duplicates, O(log n) operations

Use when: Need sorted unique elements

LinkedHashSet

Characteristics: Maintains insertion order, no duplicates, O(1) operations

Map Implementation Details

HashMap

Characteristics: Unordered, unique keys, O(1) operations, allows null key

TreeMap

Characteristics: Sorted by keys, unique keys, O(log n) operations

LinkedHashMap

Characteristics: Maintains insertion order, unique keys

ConcurrentHashMap

Characteristics: Thread-safe without locking entire map, allows concurrent read/write

import java.util.concurrent.ConcurrentHashMap;

ConcurrentHashMap<String, String> map = 
    new ConcurrentHashMap<>();
map.put("key1", "value1");
